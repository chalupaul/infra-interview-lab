# Infrastructure Automation Lab

## Setup

```bash
cd terraform
cp terraform.tfvars.example terraform.tfvars
# fill in real values or use the provided sandbox credentials
terraform init
```

## Your Task

This Terraform configuration was generated by an AI assistant and reviewed briefly before being handed to you. It provisions a Fargate ECS service behind an Application Load Balancer with autoscaling.

**It has bugs. Your job is to find and fix them.**

### Stage 1 — Fix & Extend (20 min)

1. Run `terraform validate` and `terraform plan` — note what fails
2. Fix all bugs so the configuration is valid
3. Rename variables so the code is readable without needing to consult `variables.tf` for every reference
4. Add a missing egress rule to the ALB security group

### Stage 2 — CI/CD Pipeline (20 min)

In `.github/workflows/`, create a GitHub Actions pipeline that:

1. On pull request: runs `terraform fmt -check`, `terraform validate`
2. On merge to `main`: runs `terraform plan`, then requires manual approval before `terraform apply`
3. AWS credentials must come from secrets — never hardcoded
4. **Bonus:** fail the workflow if the plan contains any resource destructions

### Stage 3 — Policy (15 min)

Complete `policy/deny_public_s3.rego` so that it:

1. Denies any `aws_s3_bucket` with a public ACL
2. Denies any `aws_s3_bucket` missing `force_destroy = false`
3. Includes at least one passing and one failing test case
4. Wire the policy check into your CI pipeline before the apply step
